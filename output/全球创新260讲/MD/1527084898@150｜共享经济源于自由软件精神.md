# 150｜共享经济源于自由软件精神

## 今日内容看点

通过阅读今天的内容，你将会掌握以下3个问题的答案：

* 共享经济概念的起源是什么？

* 什么是自由软件？

* 开放平台的价值是什么？

相信有不少人是因为滴滴或者摩拜，才第一次知道了共享经济这个概念。但很多人可能都不知道，  *共享经济其实最早萌发于一群信仰自由软件的极客 * 。而极客精神的典型代表，就是我们今天要介绍的代码托管平台 GitHub。

## 一、代码托管平台GitHub

GitHub 是全球最大的开源代码平台之一。所谓开源，就是指公开源代码。

一般来说，只有软件的原作者才拥有程序的源代码，如果公开源代码，就相当于把自己的心血免费送给了别人。但也正是因为开源，软件行业才得以迅速发展、百花齐放。

而我们今天要讲的 GitHub，可以说是开源精神的极致体现。

简单来说，GitHub 其实就是一个网站，跟其他网站没什么区别。不过，在优酷上，我们可以看到视频，而在 GitHub 上，我们看到的则是各种被分门别类管理起来的代码。就像在视频网站搜索视频一样，你可以在这里搜索、关注和转发那些你感兴趣的代码。更有意思的是，你还可以参与代码的设计，把你的想法和意见放到这些代码里。

在 IT 界，GitHub 可是扮演了举足轻重的地位。

因为程序员以男性居多，因此 GitHub 常被戏称为 IT 宅男交友社区。其实，GitHub 作为程序员的技术分享网站，无论是最前沿的技术，还是个人资料的归档管理，你几乎都能在这里找到对应的代码。

GitHub 上面的代码都被存放在一个个叫做“仓库”的地方。你可以往这个仓库里面放代码，更改它们或者将它们拷贝出来。

不过，为了仓库的安全起见，你的每次操作都会被一个日志文件记录下来，这样我们就能知道这些代码过去都经历了什么。

截至2017年4月，GitHub 已经拥有2000万的用户和5700万的仓库。当然，GitHub 之所以这么受欢迎，原因就在于它背后的自由软件和分享精神。

 *所谓的自由软件，就是说软件可以不受限制地被所有人自由使用、复制、研究、修改和分发，甚至买卖。 *

在 GitHub 上，多数仓库都是对所有人可见的，普通的程序员也可以参与源代码的维护和优化。而且，很多优秀项目都是一群来自不同国家的爱好者，为了让程序更好用，自发地贡献技术写成的。也正是因为这样，GitHub 才吸引了越来越多优秀的程序员，他们的思维在这里相互碰撞，打磨出了一个又一个优秀的程序。

那究竟这种分享精神是从何而来的呢？实际上，这种精神的起源，分别来自一位布道者，一位奠基者和背后无数的响应者。接下来，我们就来一一地了解一下。

## 二、布道者：理查德·斯托尔曼

在众多程序员心目中，理查德·斯托尔曼是神一般的存在。他始终坚信软件不应该只是被用户使用，软件的源代码还应该被用户所学习、共享和修改。

那为什么理查德·斯托尔曼会怀抱着这样的信念呢？

这还得从他的生平经历说起：

* 1953年6月，他出生在美国曼哈顿，16岁时，他第一次接触了计算机。后来因为成绩优秀，他成为了 IBM 纽约科学中心实验室的一名助手，从此跟计算机结下了不解之缘。

* 1971年，正在哈佛大学读书的斯托尔曼遇到了一个机会，那就是 MIT 的 AI 实验室正在招募“电脑黑客”来开发新的计算机系统。这个机会很合他的胃口，于是他就加入其中，成了一名专职黑客。

理查德·斯托尔曼年轻的时候加入了一个软件共享社群，社群里的所有成员都鼓励共享代码。正是这段经历，让他成为了自由软件的狂热推崇者。这种对自由软件的推崇，深刻地体现了极客精神的精髓，那就是崇尚自由，对人为限制极其不屑，以及热衷于挑战权威。

但是从20世纪80年代开始，随着微软和苹果的崛起，在利益的驱动下，越来越多人加入了专有软件系统的行列。专有软件正好跟自由软件相对，所有程序都不对外公开。

作为极客的代表，理查德·斯托尔曼认为专用软件是“邪恶”的，因为它阻止了人们之间的分享行为，让社会变得丑陋。而抵制和击败邪恶力量的方法，就是创造自由软件。

于是1983年，他宣布要创建一个自由的、完全非专有的操作系统，叫作 GNU 操作系统。两年后，他又建立了一个名叫自由软件基金会的非盈利性组织，致力于推广自由软件和促进计算机用户自由。这场自由软件运动影响了整个 IT 界。

顺便说一句：

今天我们都认为微软是了不起的企业，其实在微软刚刚兴起的时候是很受排挤的，因为当时的主流思想是自由软件，软件应该免费，不应该用来赚钱。

当然，需要强调的是，  *自由软件的含义并不是免费，而是指尊重用户的基本自由 * 。为此，理查德·斯托尔曼还专门提出了 GNU 通用公共授权条款，简称 GPL。

凡是带有 GPL 的代码，用户都能享受以下3点主要权利：

* 不管出于什么目的，所有人都拥有运行这个程序的自由。

* 所有人都拥有任意复制这些代码的自由。

* 在得到源代码的前提下，所有人都拥有改进程序，并公开发布改进的自由。

不过出于种种原因，理查德·斯托尔曼的自由软件基金会并没有开发出 GNU 操作系统的核心部件，也就是程序员们经常说的内核。至于内核的概念，你可以把它看作是一个操作系统的大脑，操作系统之所能够运转，最离不开的就是它。

## 三、奠基者：林纳斯·托瓦兹

而这个内核部分的开发者，正是我们要讲的第二位奠基人物，名叫林纳斯·托瓦兹，他就是后来的“Linux 和 Git 之父”，和理查德·斯托尔曼一样，林纳斯·托瓦兹也是一个技术狂热者。

当 Linux 内核完成后，他立刻分享了内核的所有源代码。按他的说法，如果全球各地的黑客都来动手修改源代码，肯定能够带来一场开放的协作，从而创造出真正出色的软件。

果然，Linux 内核的公布吸引了铺天盖地的志愿合作，无数协作者对它的系统进行了改进。而且只要有漏洞出现，就会有来自全世界的程序员动手修补。正是基于 Linux 内核，后来诞生了 Windows 和 iOS 系统。

令人惊讶的是，林纳斯·托瓦兹最早公布 Linux 的内核程序时只有1万行代码，而到了2009年，这个数字变成了3.24亿行，而且这些开发全都是免费的。要知道，如果用传统方式开发，研发费用可能已经达到80亿美元左右。

这也正是我们刚才提到的第三个关键词，自由软件和分享精神的来源之一：无数的响应者。

后来，随着 Linux 内核的功能增加，代码数也在不断攀升。这样一来，不但不方便大家协作，还给版本控制添了不少麻烦。

我来稍微解释一下版本控制的意思：

一份代码随着项目的进展，可能会出现不同的版本。这时我们就得对不同版本进行管理和控制，而最简单的方法就是一个版本一个文件夹。可是随着版本数的增加，文件夹会越来越多，管理和存储成本也会跟着增加。

为了解决这个问题，林纳斯·托瓦兹开发出了一套版本控制系统，并把它命名为 Git。

每当代码改动出现新的版本后，系统就可以对新版本拍一个快照，然后根据时间等信息生成一段唯一的序列。这样一来，程序员们就可以使用 Git 的日志文件来观察和追溯每一次版本的变动了。

更方便的是，如果想跳转到某一个特定的版本，你只需在日志里面找到对应的序列，向 Git 发出请求就可以实现。至于存储成本的问题，因为快照只包含了变动的信息，没有改变的信息是不会放进去的，自然也就大大缩减了储存成本。

这套系统简单易上手，所以内核的开发者在做其他项目时，也渐渐地开始使用它。于是，GitHub 就这样一传十、十传百地应运而生了。

程序员们开始把自己的项目上传到 GitHub 上面进行托管，并希望社区里的其他人也一起来参与项目的开发。后来有非常多软件都是依靠这种共享方式才取得了成功。

## 四、程序员的Facebook

其实早在 GitHub 之前，就已经有不少代码托管的网站，但是大多数网站更像是一个代码下载网站，往往代码出问题了以后只能自己修修补补，想要优化代码，还得加入特定的组群才行。

另外，从技术上来说，也是 GitHub 更胜一筹，大多数代码托管平台都采用中心化版本控制，所有的更改都要立刻同步到服务器上，一旦断网或者发生错误更改，就要花大量的时间恢复到之前的版本。而 Git 采取的是分布式管理系统，你可以在本地完成修改后，再同步到远程主机。

除了刚才提到的这些优势，GitHub 还被称作是程序员的 Facebook。

在这里，所有开发者都可以关注某个仓库，并接收到它的实时信息。同时，每个仓库又都有一个提问区，用户可以把自己遇到的问题发布到上面，然后等待作者或者其他用户来解答。这样就等于建立了用户和作者，以及用户和用户之间的互动。

这种管理方式吸引了许多著名的极客，他们纷纷把自己的项目放到 GitHub 上，而他们的粉丝也跟着转战到了这里。就像网络经济一样，一旦越来越多的节点加了进来，GitHub 的影响力就会呈现指数增长，还会进一步迫使其他项目加入。

现在，包括谷歌、Facebook、Twitter、微软在内的业界巨头都将自己的开源项目托管给了 GitHub。GitHub 的价值得到了广泛认可，2015年，它的估值就已经达到了20亿美元。

而在 GitHub 提供的支持下，也经过接近10年的发展，数以万计的优秀项目在一群具有极客精神的程序员手中大放光彩。比如最近非常火爆的 AI 框架 TensorFlow、Keras等等。所以，开放的力量真的不容小觑。

## 今日内容小结

今天我们介绍了共享经济的先驱 GitHub，希望你能掌握：

第一，GitHub 的成功核心就在于超越金钱回报的精神鼓励。就像林纳斯说的那样：“金钱并不是最强大的动力。人们在激情的驱动下，在享受快乐时所做的工作是最出色的。”

第二，共享经济的本质是一个更为宽泛的理念，它的核心离不开四个方面，包括协作、社交、开放和消费者意识。就像 GitHub 的成功离不开开发者的线上协作、互动、代码开源以及对开源软件坚定的支持态度。

第三，对于很多人来说，开放意味着手中来之不易的资源面临着控制权和价值的流失，但实际上，开放平台吸引人才创造的价值要远远超过它所带来的问题，我们说，这就是开放的力量。
